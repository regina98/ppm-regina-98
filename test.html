GITHUB

GitHub adalah layanan penginangan web bersama untuk proyek pengembangan perangkat lunak yang menggunakan sistem pengontrol versi Git dan layanan hosting internet. Hal ini banyak digunakan untuk kode komputer. Ini memberikan kontrol akses dan beberapa fitur kolaborasi seperti pelacakan bug, permintaan fitur, manajemen tugas, dan wiki untuk setiap proyek.

Hal penting dalam Github


1. git init -> untuk menginisialisasi penyimpanan Git yang kosong di /.git/
2. git status -> utk cek isi folder (kalau merah brrti ada file yang blm di add)
3. git add blabla.txt -> menambahkan file text ke staging area, bisa di ubah
4. git commit -m " " -> -m itu message, kalau sudah commit tdk bisa d ganti. kalau mau ganti di add lagi dan ke replace. "" keterangan di Github 5. git add ' ' -> menambahkan semua file txt
6. git log -> melihat apa saja yang sudah di commmit
7. git remote -u origin https://bla.com/ -> memasukkan apa saja yg di lokal ke server
8. push -> 1 file dimasukkan ke github
9. origin -> 1 folder dimasukkan ke github
10. -u -> digunakan Git untuk mengingat parameter, jadi saat akan menjalankan git push, git tau apa yg hrs dilakukan
11. git pull origin master -> menyamakan apa yang sdah ditambahkan ke server oleh orang lain sehingga file di local sama dengan file di server
12. diff -> utk melihat perbedaan dari commit yg lalu
13. git diff head -> utk memeriksa perbedaan di commit yang paling baru
14. git diff --staged -> utk cek posisi file yang barusan di ubah
15. git reset -> utk mengunstage file di stage
16. git checkout -- octocat.txt -> utk mengembalikan perubahan seperti commit terakhir
17. branch -> copy-an dr code yg kemudian digabungkan ke master branch.
18. git branch clean_up -> membuat branch bernama clean_up
19. git checkout nama -> utk switch branches
20. git rm '*.txt' -> remove semua file txt sepenuhnya
21. git commit -m "Remove all the cats" -> utk commit kalau sdh hapus file
22. git checkout master -> utk kembali ke master branch
23. git merge clean_up -> menggabungkan perubahan dari clean_up branch ke master branch
24. git branch -d (branch_name) -> utk menghapus branch
25. git push -> push semua yang sudah dikerjakan ke remote repository

    
JQuery Mobile

jQuery Mobile framework takes the "write less, do more" mantra to the next level: Instead of writing unique applications for each mobile device or OS, the jQuery mobile framework allows you to design a single highly-branded responsive web site or application that will work on all popular smartphone, tablet, and desktop platforms.

Hal penting dalam Jquery Mobile


1. Sistem yang mencatat setiap perubahan terhadap suatu file jadi bisa kembali ke salah satu versi dari file tersebut
2. version control system lokal -> bisa mengelola revisi dari banyak file,perubahan hanya pada data lokal (contoh: RCS) version control system terpusat -> apabila ada perubahan, yang ada di server juga ikut berubah version control system terdistribusi -> apabila ada perubahan di lokal harus di commit terlebih dahulu agar terjadi perubahan di server
3. download git, membuat akun + repository, buka git bash.
4. Commited -> data tersimpan pada lokal Modified -> sudah melakukan perubahan pada berkas tapi belum melakukan commit Staged -> sudah menandai berkas yang telah diubah pada versi yang sedang berlangsung yang kemudian dilakukan commit
5. git status -> utk mengetahui status dari repository lokal git init -> utk membuat repository pada file lokal yang nantinya ada folder.git git add -> menambahkan file baru pada repository yang dipilih git commmit -> menyimpan perubahan yang sudah dilakukan git push -> mengirimkan perubahan file setelah di commit ke remote repository

    
Angular JS

Angular JS merupakan framework javascript open souce yang dirilis oleh google pada tahun 2009. Konsep dari Angular JS adalah meningkatkan fungsi dari HTML untuk membangun web app. Bayangkan awalnya HTML hanya digunakan untuk membuat halaman website statis dan kini bisa berfungsi untuk membuat web app dengan menggunakan Angular JS.
